Ik cc hiervoor even Bart Koppe, want ik heb begrepen dat hij dit jaar ook betrokken is bij het Medialab.

Wat betreft het netwerk: ik heb helaas nergens gedocumenteerd hoe het precies in elkaar steekt. Maar ik heb eea. kunnen construeren uit mails van voorgaande jaren. Op zich niets nieuws voor jou denk ik, want die mails heb jij ook allemaal gehad.

Westergas levert een uplink daar in de hoek bij het systeembeheer hok. Je prikt vanaf daar een kabel in de Cisco router. Ik meen dat er stickers opzitten waar de uplink in moet. Uit mijn hoofd heeft de router aan de buitenkant IP-adres 192.168.16.10, en ze NAT’en alles vanaf 77.243.166.220 door naar dat interne IP-adres. Achter die router sluit je een HP switch aan, en daarachter weer de Cisco wireless controller. Let op dat je die controller niet direct in de router prikt, want dan kom je niet bij de management interface!

Aan de binnenkant zijn er deze VLAN’s:

VLAN10: 172.16.10.0/24, filtered internet, wired
VLAN16: 172.16.16.0/24, NOC wired & wireless - alle switches en APs zitten ook in dit VLAN
VLAN20: 172.16.20.0/24, unfiltered unternet, wired
VLAN30: 172.16.30.0/24, niet gebruikt
VLAN40: 172.16.40.0/24, filtered internet, wireless
VLAN50: 172.16.50.0/24, unfiltered internet, wireless
VLAN60: 172.16.60.0/24, WLC-4402 management

Vorig jaar hebben we de SSID’s van de draadloze netwerken generiek gemaakt, dus zonder jaartal erin.

Dan nog wat wachtwoorden e.d.:

Code slot netwerkhok:           142
Cisco username/password:        Cisco/Cisco
WPA key Cinekid-N:              C1sc0C1sc0
WPA key Cinekid-U:              joostjoost

En dan zijn er nog deze IP-adressen:

Cisco 2811 (router):            172.16.16.1 (bereikbaar via telnet)
Wireless controller:            172.16.60.2 (bereikbaar via https)

Ik kan je helaas niet exact(er) zeggen welke kabel je waarin prikt, maar heel moeilijk is het nooit. Komen jullie vast wel uit...

> Instructies:
>
> - Boot de encoder computers van een Linux livecd
> - Partitioneer de disk als volgt: /dev/sda1 (ext4), /dev/sda2 (swap)
> - Maak de filesystems aan: mkfs.ext4 /dev/sda1; mkswap /dev/sda2
> - Mount het nieuw aangemaakte ext4 filesysteem en pak de .tar.gz hierin uit
> - Herstel de bootloader (chroot /media/sda1; update-initramfs, update-grub, grub-install)
> - Als het goed is boot je nu een Xubuntu desktop omgeving. Je kan inloggen met gebruikersnaam ‘mtp’ en wachtwoord ‘koekje123’.
> - Alle conversie scripts staan in /usr/local/sbin
>
> Andere zaken die in dat Linux image zijn geregeld zijn:
>
> - Een samba server, zodat men de werkjes kan uploaden naar een share
> - Een script voor het uploaden van de geconverteerde werkjes naar de Cinekid webserver
> - Een backupscript dat al het materiaal op de externe USB-disk zet
> - Een NFS-export van de ene encoder naar de andere
>
> Het is lastig om dat allemaal uit te tikken, maar iemand met een beetje linux kennis zal hier weinig moeite mee hebben.

>> Zaken die belangrijk zijn om te regelen zijn:
>>
>> - Afspraken maken over het formaat waarin het bronmateriaal wordt
>>   aangeleverd, en de naamgeving van de bestanden. Dat eerste is
>>   belangrijk omdat sommige (proprietary) codecs niet door ffmpeg worden
>>   ondersteund. Dit moet dus vooraf voor ieder werkje getest worden!
>> - Afspraken met Mieg over toegang tot de Cinekid webserver (ik leverde
>>   ieder jaar een public ssh key aan), een overzicht van de werkjes en de
>>   naamgeving. Die naamgeving is voor Mieg ook erg belangrijk zodat hij
>>   kan voorbereiden dat werkjes automatisch op de website verschijnen.
>>
>> Wat betreft de inrichting van die Linux desktop heb ik nog een paar
>> tips:
>>
>> - Zorg dat je de scripting zo maakt dat je (per desktop) even veel
>>   instances van ffmpeg start als het aantal cores dat je tot je
>>   beschikking hebt.
>> - Zorg daarnaast dat je niet meerdere keren tegelijk hetzelfde file gaat
>>   converteren. Ik deed dat door voor ieder file een .lock file neer te
>>   zetten zodat je kon zien dat die al geconverteerd werd. Als het
>>   converteren faalde moet je uiteraard de .lock weer weghalen.
>> - Upload pas een file naar de server als deze langer dan 5 minuten
>>   geleden voor het laatst is gewijzigd (mtime). Zo voorkom je dat je
>>   files op de webserver neerzet die nog niet voltooid zijn.
>>
>> Hoewel ik ieder jaar de Linux installatie opnieuw voorbereidde en de
>> scripting grotendeels opnieuw schreef heb ik nog wel wat liggen. Ik zal
>> jullie een downloadlink toesturen van:
>>
>> - cinekid-encoder1.tar.gz; een tarball van een Linux installatie. Als je
>>   deze wilt neerzetten moet je booten vanaf een Linux live omgeving, de
>>   disk partitioneren, deze tar uitpakken en de bootloader herstellen.
